Package: org.grails.datastore.gorm.mongo.simple

[Groovy] Class EnumType

      • org.grails.datastore.gorm.mongo.simple.EnumType 

      • @groovy.transform.CompileStatic
class EnumType
extends AbstractMappingAwareCustomTypeMarshaller<java.lang.Object, org.bson.Document, org.bson.Document>

        A custom type for persisting Enum which have an id field in domain classes. For example: To save identity instead of string in database for field type. 

              class User {
          UserType type
          String name
      }

      enum UserType {
          A(1), B(2)
          
          final int id
          UserType(int id) {
              this.id = id
          }
      }

          • private void putValueToProperPlace(PersistentProperty property, java.lang.String queryKey, PropertyCriterion criterion, org.bson.Document nativeQuery)

            For custom user types, GORM return an empty map where our custom value needs to be inserted. For example: If our code is something like this: 

                  
          User.withCriteria {
              or {
                  eq("name", "admin")
                  eq("status", UserStatus.ACTIVE)
              }
              eq("foo", "bar")
          }
            Then the query we receive will be like: 
                      [$and: [[$or: [["name": "admin"], [:]]], ["foo": "bar"]]
            Now we have to place value of status to the blank field. This method searches that empty place and put the value to the right place.
